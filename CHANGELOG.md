# Changelog

## v0.1.0
- 完成 Spring Boot 项目基础搭建
- 完成用户模块 CRUD
- 完成商品模块 CRUD
- 完成数据库购物车模块
- 完成订单基础功能
- 接入统一返回结果与统一异常处理

## v0.2.0
- 接入 Redis
- 实现 Redis 购物车
- 支持修改 Redis 购物车数量与删除商品
- 为 Redis 购物车增加过期时间续期
- 实现商品详情缓存
- 实现商品列表缓存
- 增加空值缓存，防止缓存穿透
- 商品新增、修改、删除时同步删除缓存

## v0.3.0
- 接入 RabbitMQ
- 创建订单延迟队列、释放队列与交换机
- 解决 RabbitMQ 交换机和队列未自动声明的问题
- 下单成功后发送延迟取消订单消息
- 监听释放队列，自动取消超时未支付订单
- 对已支付订单增加状态校验，避免误取消
- 规范 RabbitMQ 模块日志与状态常量
- 将延迟消息从裸订单 ID 升级为对象消息

## v0.4.0

## [v0.6.0] - 2026-04-15

### Added

* 新增订单支付方法 `payOrder`，限制订单只能从 `UNPAID` 状态变更为 `PAID`
* 新增订单状态条件更新能力，支持基于旧状态进行原子流转控制
* 增加商品缓存删除逻辑，在库存变更后清理商品详情缓存和商品列表缓存

### Changed

* 优化取消订单流程，改为先将订单状态从 `UNPAID` 原子更新为 `CANCELLED`，再执行库存恢复
* 优化下单流程，继续使用数据库条件扣库存作为最终防超卖手段
* 统一订单状态流转逻辑，支付、取消、超时关单均基于条件更新控制
* 商品库存变化后同步删除 Redis 缓存，避免商品详情和商品列表返回旧库存

### Fixed

* 修复重复取消同一订单时可能导致库存被重复恢复的问题
* 修复支付与取消并发场景下订单状态可能被重复修改的问题
* 修复库存变化后 Redis 商品缓存未及时失效导致的数据不一致问题

### Notes

* 订单创建状态建议统一使用 `OrderStatusConstant.UNPAID` 常量，避免业务代码中继续使用硬编码状态值
* 商品缓存删除应放在库存实际扣减成功后执行，而不是放在库存预检查阶段

## [v0.7.0] - 2026-04-15

### Added
- 新增 BCrypt 密码加密配置，统一提供 PasswordEncoder

### Changed
- 用户注册时改为加密存储密码，不再保存明文密码
- 用户登录时改为使用 PasswordEncoder 进行密文校验

### Fixed
- 修复用户密码明文存储带来的安全风险

## [v0.7.1] - 2026-04-15

### Added

* 新增 `UserVO` 作为用户信息返回对象，用于接口响应与数据库实体解耦

### Changed

* 用户注册时改为使用 BCrypt 对密码进行加密后再存储
* 用户查询接口改为返回 `UserVO`，不再直接返回 `User` 实体
* 优化用户信息返回结构，移除接口响应中的密码字段

### Fixed

* 修复用户接口直接返回实体对象可能导致密码字段泄露的问题
